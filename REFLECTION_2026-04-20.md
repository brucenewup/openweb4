## ⚠️ 遇到的问题与解决

### 问题 1: Git Push SSH 配置错误
**现象：**
```
kex_exchange_identification: Connection closed by remote host
Connection closed by 127.0.0.1 port 7890
```

**原因：**
- 远程仓库配置：`git@github-brucenewup:brucenewup/openweb4.git`
- SSH 配置缺失：`~/.ssh/config` 中无 `github-brucenewup` 配置
- 使用了错误的密钥：`id_rsa`（clarkjia）而非 `id_ed25519_openweb4_bruce`（brucenewup）

**解决方案：**
1. 添加 SSH 配置到 `~/.ssh/config`
2. 使用正确的密钥：`id_ed25519_openweb4_bruce`
3. 测试连接：`ssh -T git@github-brucenewup`
4. 成功推送

**教训：**
✅ **遇到 SSH 问题，先检查：**
1. SSH 配置是否存在
2. 密钥是否正确
3. 用户权限是否匹配

---

### 问题 2: 端口 8080 被占用
**现象：**
```
Port 8080 is already in use
```

**解决方案：**
```bash
lsof -ti:8080 | xargs kill -9
```

**教训：**
✅ **启动服务前先清理端口，避免冲突**

---

### 问题 3: 浏览器工具 npm 错误
**现象：**
```
npm ERR! ENOTEMPTY: directory not empty
```

**解决方案：**
- 改用 curl 测试 API
- 避免依赖不稳定的工具

**教训：**
✅ **关键测试不要依赖单一工具，准备备用方案**
